Having selected the features to be included in the formula, we have a formula with 10 features. To split the data were going to set a seed and set the partition rate on 80/20 for training and testing.

set.seed(2403)
ml_data <- pre_ml_data %>%
  select(pts, fgm, minutes, ftm, countSeasons, tov, fg2m, dreb, ast, stl, pctSalaryTotal)

trainIndex <- createDataPartition(ml_data$pctSalaryTotal, p = .8, list = FALSE, times = 1)

training_set <- ml_data[ trainIndex, ]
test_set <- ml_data[ -trainIndex, ]

Once we have the data splitted, we have to set our model. We chose a Parallel Random Forest, provided in the caret package because is the one of the most accurate machine learning algorithms available nowadays.

We set a repeated cross-validation control to the model and also a grid with 3 different mtry values.

fitControl <- trainControl(
  method = "repeatedcv",
  number = 10,
  repeats = 3
)

tune_grid <- expand.grid(.mtry = c(2,6,10))

model_fit <- train(pctSalaryTotal ~ .,
                   data = ml_data,
                   method = "parRF", # Parallel Random Forest
                   trControl = fitControl,
                   tuneGrid = tune_grid,
                   na.action = na.exclude)
model_preds <- predict(model_fit, test_set)

As we can see in the model report, the best RMSE obtained is 0.001329462.

We have to take into account the response variable is not the salary but the % of salary compared with the total of NBA salaries. So if we multiply the MAE by this amount, we get the MAE on the original variable of 1542394. Taking into account our dataset only contains data of one season and the are a lot of factors affecting the players salaries which are not part of the game itself, were really satisfied with the performance of our model.

With this results we could provide a quite accurate prediction of their salaries to players from other leagues who want to move to the NBA.
